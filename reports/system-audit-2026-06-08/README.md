# System Audit 2026-06-08

Bu klasor, sistemin koddan yapilan canli incelemesi icin olusturuldu.

Kural: onceki `.md` rapor/memory dosyalari analiz kaynagi olarak okunmayacak. Bulgular repo dosya yapisi, kaynak kod, konfigler, testler ve calistirilan komutlardan uretilecek.

## Dosyalar

- `audit-log.md`: Yapilan islemlerin kronolojik kaydi.
- `todo.md`: Audit icin kendi kontrol listem ve ilerleme durumu.
- `repo-inventory.md`: Monorepo, uygulamalar, paketler ve teknoloji yigini envanteri.
- `module-map.md`: Moduller ve baglantilarin haritasi.
- `web-analysis.md`: Web uygulamasi incelemesi.
- `admin-analysis.md`: Admin uygulamasi incelemesi.
- `mobile-analysis.md`: Mobile uygulamasi incelemesi.
- `home-analysis.md`: Anasayfa/public landing ve pazarlama yuzeyi incelemesi.
- `api-data-flow.md`: API, veri modeli, servis ve entegrasyon akislari.
- `findings.md`: Hata, eksik, risk ve teknik borc bulgulari.
- `recommendations.md`: Iyilestirme onerileri ve onceliklendirilmis yol haritasi.
- `route-inventory.md`: Web/admin/mobile route ve endpoint aileleri.
- `client-endpoint-map.md`: Web, mobile ve admin istemcilerinin API baglantilari.
- `env-inventory.md`: `.env.example`, kaynak kod ve runtime-config anahtar uyumu.
- `workspace-connector-analysis.md`: Workspace modeli ile connector/sync akislari.

## designSummary

A single-file dark "Move — Operations" admin console mockup. Fixed **250px flat sidebar** with a brand block (Raccoon mascot mark in a rounded gradient tile + Playfair "Move" wordmark + gold mono "Operations" eyebrow), a **flat 9-item nav** (Overview, Users, Moves, Analytics, Content, Providers, Support [badge "37"], Data sources, Settings — active item marked by a left gold bar), and a footer user chip ("Avery Ops · Super admin" + sign-out icon). Main area = sticky blurred topbar (Playfair page title + subtitle that change per view, a 240px search box "Search users, moves…", a bell with red dot). Nine in-page views toggled client-side (no routing):
- **Overview:** 5 KPI cards (Total users 24,318, Active moves 3,942, Services transferred 18,204, Referral revenue $42,180, Open tickets 37) with delta pills + inline SVG sparklines; a "New signups" 8-week bar chart (+18.4%); a "Live ops / Needs attention now" rail (High-risk moves, Provider failures, Urgent tickets, Sync errors with emoji icons); a "Recent moves" table (Customer / Route / Move date / Progress bar / Risk pill).
- **Users:** table (Name / Email / Plan / Moves / Status) with Active/Trial/Churned status pills, PRO/Free plan pills.
- **Moves:** 4 stat tiles (Active 3,942 / High risk 312 / Completing this week 486 / Avg progress 58%) + moves table.
- **Providers:** "Add provider" CTA + card grid (emoji icon, name, type, Operational/Degraded/Down status dot, Requests-today + Avg-latency).
- **Support:** queue header ("37 open · avg first reply 1h 12m") + ticket rows (priority dot, subject, user · #id, Urgent/High/Normal pill, age).
- **Data sources:** card grid (FEMA, EPA AirNow, NWS, US Census, WalkScore) with Healthy/Syncing/Error status, last-sync, coverage %.
- **Analytics:** 4 stat cards (Conversion 9.0%, Retention 90d 78%, Avg services/move 4.6, NPS 64) + Conversion funnel bars + Revenue-by-source list + Top-routes bars.
- **Content:** "New guide" CTA + guides table (Title / Status / Views / Author / Updated) with Published/Draft/Review pills.

## Detail notes

**Brand / rebrand (admin-01, admin-18).** Every brand string in the admin is "LocateFlow" (`sidebar.tsx:277-278,332`, `layout.tsx:41,46,90`, plus the `next/font` comment at `layout.tsx:11` and theme-storage key `locateflow-admin-theme`). The design rebrands to "Move" with the "Operations" eyebrow retained. Team emails in the mock are `@move.app`. The raccoon mascot is shared with the current build (already `/logo-mark.svg`), but the design renders it as a live `Raccoon` component with `mood/head/mask/ear/eye/pupil` props (line 30) and a separate `Raccoon.dc.html` exists in the bundle — so the mascot is **kept and elaborated**, not new. Rebrand is a product-scope decision (admin-01 decisionNeeded).

**Theme (admin-02, admin-03, admin-20).** Important nuance: the mock's **background** matches the brief's navy `#070B14`, but its **accent default is still Gold `#CBA45E`** (the accent prop offers Gold/Sapphire/Emerald, no teal). The brief states the rebrand accent should be **teal/green `#168E9C` / `#1C8A63` / `#2A8E66`**. So the design file itself does **not** yet encode the teal/green accent — this is a conflict to confirm: follow the brief's teal/green, or the file's Gold? Flagged decisionNeeded. The mock is also **dark-only**; current admin ships full light+dark — decide whether light mode survives the rebrand (admin-20).

**Layout / nav (admin-04, admin-05, admin-06).** The single biggest structural delta. The mock is a flat 9-item sidebar with client-side view toggling and no RBAC; the current admin is a two-tier grouped rail+panel over ~33 RBAC-gated routes across 62 pages. The mock collapses/renames a lot: "Moving Plans" → **"Moves"**, Blog+Help-Center → **"Content"**, and introduces a **"Data sources"** section (external feed health: FEMA/EPA/NWS/Census/WalkScore) that has no current admin equivalent (closest is `/connectors`, a different concept). Adopting the mock's IA wholesale would drop or fold dozens of existing routes (Subscriptions, Workspaces, Movers, Connectors, Feature-Flags, Runtime-Config, Security, Audit-Logs, etc.) — a major product decision. Most likely the mock is an **illustrative subset**, not a literal target IA; confirm scope before restructuring nav.

**New behaviors (admin-11, admin-13).** "AI briefings — Generate daily AI move summaries" toggle has no current counterpart (admin-11). The mock foregrounds a per-move **risk** model in the admin (risk pills in lists, "High risk 312" stat, Live-ops "High-risk moves") — the current admin has `/moving/at-risk` board but the main moves table lacks a risk column; partial parity (admin-13).

**Where current already matches.** The Overview view maps almost 1:1 onto the existing dashboard (KPI tiles w/ sparkline + count-up, live-ops rail, trends chart, recent-moves/users, integrations) — gap is metric labels/values and styling, not structure (admin-14). Users/Providers/Support/Analytics/Content all have current route equivalents; differences are layout/grouping/copy, not missing capability.

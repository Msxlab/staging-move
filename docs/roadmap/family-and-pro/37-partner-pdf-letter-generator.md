## Amaç

Doktor muayenehanesi, küçük bölgesel utility, lokal banka şubesi gibi **dijital adres değişikliği kabul etmeyen** partner'lar için server-side PDF mektup üretir. Kullanıcı PDF'i indirir, imzalar, posta/fax/elden teslim eder. D15 gereği lansman partner'larının büyük çoğunluğu için tek geçerli kanal budur.

`ServiceProviderAction.actionType = GENERATE_PDF` taşıyan action'lar bu generator'a yönlenir.

## Kapsam

**In scope (MVP)**
- `@react-pdf/renderer` ile server-side render (Node runtime, Edge'de değil)
- 3 başlangıç template: `doctor-office-address-change`, `utility-cancellation-notice`, `bank-address-update`
- Template variable substitution: kullanıcı adı, eski/yeni adres, hesap no, servis adı, etkin tarih, recipient (provider) adı + adresi
- TR/EN dil seçimi (cross-ref [67](./67-i18n-tr-en.md))
- Watermark: "Generated by LocateFlow on {date} — ref:{attemptId}"
- Endpoint: `GET /api/partner-actions/:attemptId/pdf`
- Web: doğrudan download (`Content-Disposition: attachment`)
- Mobile: `expo-file-system` ile cache'e indir + `expo-sharing` ile share sheet
- Audit log: PDF generation `AuditLog(action="GENERATE", entityType="PartnerPdf", entityId=attemptId)`

**Out of scope**
- E-imza (DocuSign benzeri) — Faz 2
- PDF preview iframe (Faz 2)
- Toplu PDF (10 partner için tek ZIP) — Faz 2
- Template'lerin admin UI'dan editlenmesi — kod-bazlı (cross-ref [51](./51-admin-provider-actions-crud.md): admin sadece `pdfTemplateKey` seçer)
- Faks/posta otomatik gönderim entegrasyonu (Lob, PostGrid) — Faz 2

## User stories

- As Pro user, doktor muayenehaneme adres değişikliği bildirmek için **PDF indirip imzalayıp postalayabileyim** — telefon/email kabul etmeyen partner için tek geçerli kanal.
- As Pro user, PDF'in **eski adresimi, yeni adresimi, hasta no/üye no'mu** otomatik doldurmuş olmasını isterim — el yazısı hataya açık.
- As Pro user, **TR/EN dil seçeneği** isterim — Türk doktora TR, ABD bankaya EN.
- As Pro user, PDF üzerinde **referans no görmek** isterim — partner aradığında "şu referansla gönderdim" diyebilmeliyim.
- As support agent, kullanıcı "PDF'i indirdim ama yanlış adres var" derse **hangi event'ten hangi tarihte üretildiğini** AuditLog'dan görebilmeliyim.

## Veri modeli

Yeni tablo **yok**. `ServiceProviderAction` üzerine eklenir (cross-ref [34](./34-service-provider-action-registry.md)):

```prisma
model ServiceProviderAction {
  // ... mevcut alanlar ...

+ actionType      String  @db.VarChar(40)
+   // DEEP_LINK | MAILTO | GENERATE_PDF | PHONE_CALL | IN_PERSON
+ pdfTemplateKey  String? @db.VarChar(80)
+   // örn: "doctor-office-address-change-v1"
+   // null değilse actionType=GENERATE_PDF olmalı (DB constraint değil, app-level invariant)
+ pdfLocale       String? @db.VarChar(5)
+   // "tr-TR" | "en-US"; null ise kullanıcı locale'inden türetilir
}
```

Migration: backfill yok, eski action'lar `actionType=DEEP_LINK` alır (cross-ref [36](./36-partner-deep-link-launcher.md)).

## API endpoint'leri

### Yeni

| Method | Path | Auth | Workspace ctx | Body | Response | Errors |
|---|---|---|---|---|---|---|
| GET | `/api/partner-actions/:attemptId/pdf` | session cookie | `requireWorkspaceContext` | — | `application/pdf` stream, `Content-Disposition: attachment; filename="address-change-{providerSlug}-{YYYYMMDD}.pdf"` | 401, 403 (attempt başka workspace'e ait), 404 (attempt yok), 409 (action `GENERATE_PDF` değil), 422 (template yok / variable eksik), 429 (rate limit) |
| GET | `/api/partner-actions/:attemptId/pdf/preview` | session cookie | `requireWorkspaceContext` | — | `application/pdf` inline (Pro hesabı için preview, Faz 2) | — |

**Server logic** (`apps/web/src/app/api/partner-actions/[attemptId]/pdf/route.ts`):
1. `requireWorkspaceContext(request)` (cross-ref [07](./07-api-workspace-context-helper.md))
2. `prisma.partnerSyncAttempt.findFirst({ where: { id, event: { workspaceId } } })` — workspace izolasyonu
3. Attempt → event → fromAddress + toAddress + target user yükle
4. Provider + action yükle, `actionType === 'GENERATE_PDF'` doğrula
5. Template'i `apps/web/src/lib/pdf-templates/registry.ts` lookup ile çöz
6. React PDF `renderToBuffer(<Template variables={...} />)`
7. `AuditLog.create({ action:"GENERATE", entityType:"PartnerPdf", entityId: attemptId, changes: JSON.stringify({ providerId, templateKey, locale }) })`
8. `PartnerSyncAttempt.update` → `status: OPENED`, `lastOpenedAt: now()`, `openCount: { increment: 1 }` (D7)
9. Stream buffer ile dön

**Rate limit**: 30 PDF / dakika / user (puppeteer benzeri abuse'a karşı, gerçekçi kullanım çok altında).

### Mevcut endpoint'lere etki

- `/api/partner-actions/:attemptId/launch` (cross-ref [36](./36-partner-deep-link-launcher.md)) — `actionType === 'GENERATE_PDF'` ise 308 redirect ile `/pdf` endpoint'ine yönlendirir. Tek bir "Open Action" buton handler'ı kalır.
- `/api/events/:eventId` — response'a her attempt için `availableChannel: "PDF" | "MAILTO" | "DEEP_LINK" | ...` field'ı eklenir (UI doğru butonu çizebilsin).

## Güvenlik

- [x] **Step-up auth?** Hayır per-button — D10/D19 gereği event-level challenge yeterli. Event yaratıldıktan sonra PDF indirme limitsiz.
- [x] **PII redaction?** PDF içeriği zaten kullanıcının PII'si (ad, adres, hesap no). URL'de PII yok — `attemptId` opaque cuid. Server `Content-Disposition` filename'inde provider slug + tarih, kullanıcı adı YOK.
- [x] **Audit log?** Evet — `AuditLog(action="GENERATE", entityType="PartnerPdf", entityId=attemptId, changes={providerId, templateKey, locale})`. Admin regenerate ise `action="ADMIN_REGENERATE"` + `AdminAuditLog` (cross-ref schema satır 1074).
- [x] **Rate limit?** 30/dk/user — abuse'a karşı (existing `RateLimitBucket` tablosu, schema satır ~1340).
- [x] **Permission matris?** OWNER + ADMIN + MEMBER PDF üretebilir (kendi event'leri). VIEW_ONLY salt-okur, PDF endpoint'e GET edebilir AMA event yaratamadığı için pratikte sadece başkasının event'ini görür. CHILD: kendi target olduğu attempt için PDF indirebilir (cross-ref [22](./22-child-role.md)).
- [x] **Encryption at rest?** PDF disk'e kalıcı yazılmaz (stream response). Mobile cache: `expo-file-system` cache directory app-private, OS-level encryption'a tabi. Hesap no DB'de zaten encrypted (`packages/shared/src/encryption.ts`) — render anında decrypt.
- [x] **GDPR DSAR?** PDF'ler kalıcı tutulmadığı için DSAR export'a dahil DEĞİL. Audit log entry'leri (kim ne zaman ne ürettiğini) DSAR export'a dahil — mevcut `/api/profile/export` AuditLog dump'ı zaten kapsar.

## Migration / backward compat

- `ServiceProviderAction.actionType` migration: yeni kolon, nullable→default `'DEEP_LINK'`, sonra not-null. Mevcut row'lar otomatik `DEEP_LINK` alır.
- `pdfTemplateKey` ve `pdfLocale` nullable kalır.
- Eski client'lar (cache'lenmiş JS bundle) `actionType` field'ını ignore ederse default davranış (deep-link) kullanır — graceful degradation.

## Etkilenen mevcut özellikler

## Açık sorular

1. Template versionlama: `doctor-office-address-change-v1` → `v2` geçişinde eski PDF üreten attempt'ler için ne olur? **Öneri**: registry hem v1 hem v2 tutar, action'da hangi key varsa o kullanılır (immutable). Yeni provider create ediliyorsa admin v2 seçer.
2. PDF'e QR kod ekleyelim mi (partner'ın LocateFlow'a confirmation gönderebilmesi için)? **Karar**: MVP'de yok — partner API olmadan QR sadece marketing değeri taşır, security review'a değmez.
3. Çoklu sayfa PDF (örn. 5 sayfa banka formu) gerekecek mi? React PDF destekler ama template karmaşası artar. **Karar**: MVP tüm template'ler tek sayfa; multi-page Faz 2.
4. Doğrudan postaya verme entegrasyonu (Lob API, $1–2/mektup): değer önerisi güçlü ama lansman blocker. Faz 2.
5. Kullanıcı imzasını dijital ekleyebilsin mi (Pro Plus özelliği)? Faz 3.

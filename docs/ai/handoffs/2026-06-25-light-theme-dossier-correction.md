## Web Findings

- Design tokens already had the correct source beige in `apps/web/src/styles/_tokens.generated.css` as `--bg: #EFEADF`.
- The regression was a local override in `apps/web/src/styles/globals.css`: `.light { --lf-app-bg: #FAF7F0; }`, which pulled the authenticated shell away from source beige.
- Current web dossier rows already render `lf-dossier-scene-card` and `DossierAmbient` with source-scene `data-ds-*` attributes, but the light scene styling was too washed out.
- Web dossier cache is client-side `sessionStorage` with TTL from `Cache-Control`, plus an API in-process cache. The service worker intentionally does not cache `/api/` responses.
- PDF route delegates to dossier data and uses `generateDossierReportPdf`. Existing code includes the `standard-font-data` pdfkit shim for the known standalone font ENOENT class of PDF 500.

## Mobile Findings

- Mobile `DossierAmbient` is not currently source-layout parity: it still renders a right-side decorative layer per row, not the source 82px top scene band. No mobile source was changed in this pass.

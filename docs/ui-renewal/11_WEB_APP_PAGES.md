## 0. Shared shell, design-token system & global conventions

These facts apply to (almost) every row in the tables below; the per-page cells assume them unless they note a deviation.

### Shell
- **Route-group layout:** `apps/web/src/app/(app)/layout.tsx` — server component. Auth-gates via `requireDbUserId`/`getAppGateState` (redirects to `/sign-in`), resolves `showBudget`, `showWorkspace`, `planTier`, `consumerFree`, then renders `AppShell`.
- **Shell component:** `apps/web/src/components/layout/app-shell.tsx` (`"use client"`). Composes `Sidebar` (desktop + mobile drawer variant), `Header`, `MobileNav` (bottom tab bar), `ImpersonationBanner`, `PendingInvitationsBanner`, `InstallPrompt`, an `app-shell-backdrop` decorative layer, and a skip-to-main link. Main content wrapper: `<main className="flex-1 p-4 md:p-6 pb-20 md:pb-6">` with `<div className="mx-auto w-full max-w-screen-2xl">`. Background = `style={{ background: "var(--surface)" }}`.
- **Embed mode:** `?embed=mobile` (latched in `sessionStorage` as `lf:embed-mobile`) strips all chrome and renders a bare `max-w-screen-md` padded container — for native in-app browser sessions. Renewal must preserve this stripped variant.
- **Plan class hook:** `planClass` (`plan-pro`/`plan-family`/`plan-free`/`""`) is applied to the shell root but **all three resolve to the same Sapphire `--primary`** in `globals.css` (intentional pass-through; Individual is classless). No per-tier color today.

### Token system (`apps/web/src/styles/globals.css`)
- **Dark-first.** `:root` = dark (`--surface: #121B2D`); `.light` class overrides to light (`--surface: #FFFFFF`). Theme switching is class-based (`AppearanceCard` in settings; light/dark/system).
- **`--primary` ("Sapphire"):** dark `38.53 51.17% 58.24%` (warm gold/honey HSL), light `217.38 58.56% 43.53%` (blue). Consumed as `bg-primary`, `text-primary`, `hsl(var(--primary))`, `border-primary/NN`, `ring`. **Note the primary hue flips between modes** — gold in dark, blue in light.
- **Tone ramp** (the dominant accent system): `tone-{orange,cyan,emerald,honey,sky,foil,slate,sage,amber,rose,cyan}` each with `-bg` / `-fg` / `-br` Tailwind utilities mapped to CSS vars that have **both dark and light values**, so tone usage is light+dark-safe.
- **Brand/legacy:** `--brand-orange` (dark `#CBA45E`, light `#2E5FB0`) and `--tone-orange-fg` track the brand accent; `--sage` (mint/green), `--warning`, `--destructive`, `--success`, `--muted`, `--card`, `--popover`, `--border`, `--ring`, `--accent` all tokenized.
- **Typography:** `font-display` (serif display face); `.h1` / `.h2` helpers apply the display face with a gradient `<em>` accent (cool gradient). `font-mono` used heavily for eyebrows, stats, dates.
- **Decorative:** `.app-shell-backdrop` is a fixed gradient overlay using `color-mix` over `var(--surface)`.

### Cross-cutting conventions (apply to nearly all pages)
- **Icons:** `lucide-react` throughout.
- **Toasts:** `sonner` (`toast.success/error`).
- **i18n:** **Inconsistent.** Three patterns coexist — (a) `next-intl` `useTranslations`/`getTranslations` (dashboard, addresses list/detail/new, moving list/new, notifications, services list/new); (b) inline bilingual `*_COPY = { en, es }` objects keyed by `useLocale()` (budget, settings hub, settings/export, settings/profile, settings/notifications, settings/privacy, support list); (c) **hardcoded English only** (addresses `[id]/edit`, services `[id]` + `[id]/edit`, budget `[month]`, support `[id]`, settings/connections, settings/address-changes, error, not-found). A renewal that re-theming copy must reconcile these three.
- **Card idiom:** Two visual card languages coexist — (1) the **"glass" idiom** `rounded-2xl border border-border bg-foreground/5 backdrop-blur-xl` (most list/dashboard/feature cards); (2) the **shadcn `Card`** primitive (`@/components/ui/card`) on form pages (addresses new/edit, moving new, services [id]/edit, budget [month]). Renewal must decide between these two.
- **Hardcoded color hex:** essentially none in JSX `className`s — color comes through tokens. **Exceptions (inline `style`):** `addresses-client.tsx` uses raw `hsl(var(--border))`, `hsl(var(--destructive))`, `var(--sage)`, and a `linear-gradient(90deg, var(--sage), hsl(var(--primary)))` in the health ring + transit hero; `app-shell` uses `style={{ background: "var(--surface)" }}`; budget/settings headers use `bg-[linear-gradient(135deg,hsl(var(--background))_0%,hsl(var(--muted))_100%)]` arbitrary values. All token-derived, so theme-safe, but they bypass utility classes.
- **Brand/logo elements:** No logo image on inner pages. Brand presence = the Sapphire/`--primary` accent + tone-orange brand accent + the "X command" mono eyebrow pattern (Address/Moving/Money/Account/Support command). The logo lives in `Sidebar`/`Header` (shell), not page bodies.
- **Responsive:** Tailwind breakpoints `sm/md/lg/xl/2xl`. Common patterns: single-column → `md:`/`lg:` multi-column grids; dashboard uses `lg:grid-cols-3` two-column layout; `MobileNav` bottom bar + `pb-20 md:pb-6` clearance on mobile.
- **States:** Loading via `DashboardSkeleton`/`CardSkeleton`/`LoadingSpinner` (`@/components/shared/loading-state`) or inline `Loader2 animate-spin`; empty via `EmptyState` (`@/components/shared/empty-state`, several with `RaccoonReading` illustration) or inline empty blocks; errors via toast or inline `bg-destructive/10` banner.

## 9. Cross-page renewal flags (summary of what a redesign must reconcile)

1. **i18n fragmentation** — three patterns (next-intl / inline `*_COPY={en,es}` / hardcoded English). Hardcoded-English pages: addresses `[id]/edit`, services `[id]` + `[id]/edit`, budget `[month]`, support `[id]`, settings/connections, settings/address-changes, `(app)/not-found`. A theme/copy renewal should unify on one mechanism.
2. **Two card idioms** — glass (`bg-foreground/5 backdrop-blur-xl`) vs shadcn `Card`. Form pages lean shadcn; lists/dashboard lean glass. Pick one for visual coherence.
3. **Inline `var()`/arbitrary-value styling leaks** — `addresses-client.tsx` (ring + transit gradient using raw `hsl(var(--…))`/`var(--sage)`), app-shell `var(--surface)`, budget/settings `bg-[linear-gradient(135deg,…)]`. All token-derived (theme-safe) but bypass utilities; refactor targets.
4. **Plan-tier theming is dormant** — `plan-pro`/`plan-family`/`plan-free` classes exist but all map to Sapphire `--primary`. A renewal could (re)activate per-tier accents here.
5. **Dark-first token model** — `:root` is dark; `.light` overrides; `--primary` hue flips (gold↔blue) between modes. Any new palette must supply both.
6. **Tone ramp is the real accent system** — most semantic color comes from `tone-{orange,cyan,emerald,honey,sky,foil,slate,sage,amber,rose}` `-bg/-fg/-br`, plus many JS color maps (`categoryColors`, `statTone`, `statusBadge`, `MODE_BADGE`, `presentationFor`). These maps are where a reskin's semantic colors live.
7. **"X command" eyebrow + serif `.h1`/`.h2`** is the consistent brand-voice device on hero headers (Address/Moving/Money/Account/Support command). No page-body logo; brand identity = accent + typographic device.
8. **Monetization/compliance UI** — affiliate CTAs + disclosures on providers, services, and moving/plan (flag-gated `OFFERS_*`); must keep disclosure visible after reskin.

Initial code-only inventory

Repository shape
- Monorepo: pnpm + turbo.
- Apps: apps/web, apps/admin, apps/mobile.
- Shared packages: packages/shared, packages/db, packages/connectors.
- Operational layers: scripts, docker, root deployment/config files.
- Audit source rule applied: repository .md and memory-like files were not read as audit inputs.

Framework/runtime inventory from package manifests
- Root: Node 22.x, pnpm 9.15.0, turbo 2.9.14.
- Web: Next 16.2.6, React 19.2.6, Stripe SDK, Resend, Upstash Redis/rate limit, Sentry, TanStack Query, Playwright/Vitest.
- Admin: Next 16.2.6, React 19.2.6, Prisma client, Stripe SDK, Resend, TipTap, Sentry, Vitest.
- Mobile: Expo 55, React Native 0.83.6, Expo Router, expo-iap, expo-notifications, expo-secure-store, expo-local-authentication, Expo Updates, Vitest.
- Shared: TypeScript + zod, exported to web and React Native via separate entrypoints.
- DB: Prisma 5.22, MySQL-oriented migrations, seed scripts.
- Connectors: connector core framework plus USPS connector.

Immediate audit implications
- Web and admin are broad enough that route coverage must be audited by module, not only by app.
- Payment/subscription logic spans web APIs, admin APIs, shared billing/acquisition, Prisma schema, mobile IAP, and webhooks.
- SEO/GEO has dedicated code paths: sitemap, robots, llms.txt, llms-full.txt, JSON-LD, public pages, programmatic city/state pages.
- Admin-to-web coupling is direct through shared Prisma tables and some web internal/admin alert endpoints, so authorization and data integrity must be checked across both apps.
